# ntfy-zero-inbox

Кроссплатформенный (Linux, macOS; в перспективе iOS) трей-инбокс для
[ntfy](https://ntfy.sh): собирает уведомления от автоматизаций, раскладывает по
каналам (топикам) и **кластеризует сообщения внутри канала локальной моделью на
CPU** — чтобы одинаковые/похожие алерты схлопывались в темы, а не забивали ленту.

Цель — zero-inbox: видно, что нового по каждому каналу, «сколько там ещё» внутри
темы, и всё отмечается обработанным.

> Прототип на нативном SwiftUI (только macOS) остался в ветке `legacy-swift`
> и теге `swift-prototype`.

## Стек

| Слой | Технология |
|---|---|
| Каркас | **Tauri v2** (десктоп + мобильные таргеты) |
| Фронтенд | **Svelte 5** + TypeScript + Vite |
| Бэкенд | Rust: `reqwest` (стрим ntfy), `rusqlite` (хранилище) |
| Эмбеддинги | `fastembed` — all-MiniLM-L6-v2, ONNX, CPU |
| Кластеризация | онлайн-жадная по косинусной близости центроидов |
| Трей-попап | `tauri-plugin-positioner` |

## Как работает

1. Rust-бэкенд держит стрим `GET /<topics>/json` с переподключением и догрузкой
   пропущенного через `since`.
2. Каждое новое сообщение → эмбеддинг → сравнение с центроидами кластеров **этого
   канала**; похоже (cosine ≥ 0.6) — в существующую тему, иначе — новая.
3. Всё пишется в SQLite. Фронт (попап из трея) показывает каналы → темы →
   сообщения, с отметкой «обработано».
4. Локальная модель (~23 МБ) скачивается один раз в фоне; пока не готова —
   работает резервный эмбеддер (хеширование токенов), приложение сразу живое.

## Требования

- **Rust** (stable) + системные зависимости Tauri:
  https://tauri.app/start/prerequisites/
  (на Linux — webkit2gtk, libayatana-appindicator и пр.)
- **Node.js** 18+.

## Запуск

```bash
make dev        # поставит JS-зависимости и запустит дев-режим
```

Первая сборка Rust (fastembed/ONNX) долгая — это норма. Дальше:

1. Клик по иконке в трее → откроется попап.
2. ⚙ → впиши **сервер**, **токен** (если закрытый) и **топики** через запятую →
   «Применить и переподключиться».
3. Проверка:
   ```bash
   curl -H "Title: Тест" -d "Привет" https://твой-хост/alerts
   ```

Релизный бандл: `make build`.

## Про iOS (перспектива)

`fastembed`/ONNX Runtime под iOS из коробки не собирается, поэтому в `Cargo.toml`
он исключён для `target_os = "ios"`, а эмбеддер спрятан за трейтом `Embedder`.
Варианты на будущее: перейти на `candle` (чистый Rust) или считать эмбеддинги на
сервере. Сейчас цель — Linux + macOS.

## Структура

```
src/                     # Svelte-фронтенд (попап)
  App.svelte             # каналы → темы → сообщения
  lib/
    api.ts               # обёртки над invoke + подписка на события
    Settings.svelte      # сервер / токен / топики
    types.ts, time.ts
src-tauri/               # Rust-бэкенд
  src/
    lib.rs               # сборка приложения, трей, попап, фон-задачи
    ntfy.rs              # стрим ntfy + реконнект + обработка сообщений
    cluster.rs           # эмбеддеры (fastembed/резерв) + онлайн-кластеризация
    store.rs             # SQLite: схемы и запросы
    models.rs            # типы сообщений/настроек/DTO
    state.rs             # общее состояние
    commands.rs          # команды для фронтенда
  tauri.conf.json, capabilities/
```
